## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Description | Severity | Prevention |
|---------|-------------|----------|------------|
| Stale dev server | `npm run dev` loads server code once at startup. Most "it does nothing / 404" symptoms this session were the running server lacking newly-added endpoints (`/legitimate`, `/scope`, `/ai-control`, prompt changes). | blocking | After ANY change under `companion/src/**`, restart: stop the process on `127.0.0.1:4773`, then `cd companion && npm run dev`. Dashboard now shows "restart the companion server" on 404 — keep that. |
| Over-strict AI parsing | A single unexpected model value (IOC type `"malware"`) or a markdown ```json fence rejected the ENTIRE synthesis, losing all findings/IOCs. | blocking | Parse with `extractJsonText` before `JSON.parse`; keep schema enums lenient with `.catch(fallback)` (see `analysis/responseSchema.ts`). Never tighten these back to strict. |
| Extraction guessing finding ids / capture time | Per-window extraction can't know finding ids (synthesis makes them) and must not stamp events with the screenshot capture time. | advisory | Extraction prompt omits `relatedFindingIds`; synthesis back-links via `relatedEventIds`. Event timestamps come from the artifact's own time column. |
| Merge vs replace in synthesis | Merging findings across synthesis runs left stale/out-of-scope conclusions. | advisory | `synthesize()` REPLACES findings/IOCs/MITRE (clears before merge); preserves forensic timeline + threads. Required for scope/legitimate exclusion to actually remove things. |
